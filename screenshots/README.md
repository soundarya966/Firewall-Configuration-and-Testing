# Screenshots Documentation

This folder contains screenshots taken during the firewall configuration and testing process using UFW on Linux. Each image demonstrates a specific step of the task.

---

### 01_ufw_status_verbose.png
Shows the current status of the UFW firewall using `sudo ufw status verbose`.  
Confirms that the firewall is active, default incoming traffic is denied, outgoing traffic is allowed, and logging is enabled.

---

### 02_allowed_ports.png
Displays the list of allowed firewall rules.  
Confirms that essential service ports such as SSH (22), HTTP (80), and HTTPS (443) are permitted.

---

### 03_blocked_port_21.png
Demonstrates the blocking of port 21 (FTP).  
Verifies that unused or insecure ports are denied to reduce the attack surface.

---

### 04_blocked_malicious_ip.png
Shows a firewall rule blocking a suspicious IP address.  
Prevents unauthorized or malicious access from the specified IP.

---

### 05_ufw_logs.png
Displays firewall logs generated by UFW.  
Shows blocked traffic entries, confirming that firewall rules are being enforced and monitored.

---

### 06_connectivity_test.png
Shows connectivity testing results.  
Confirms successful access to allowed services and failure to connect to blocked ports.

---

## Summary
These screenshots provide visual proof of firewall rule configuration, traffic filtering, logging, and security enforcement as part of Task.
